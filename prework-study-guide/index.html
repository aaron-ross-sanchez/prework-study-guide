HTML

Hypertext Markup Language, HTML gives the content structure.

Basics

  • An element refers to the entire element from opening tag to closing tag.
  • A tag refers only to what appears inside the angle brackets.
  • The <opening> tag begins the element. They start with a left-angle bracket and end with a right-angle bracket.
  • The </closing> tag ends the element. They also start with a left-angle bracket and end with a right-angle bracket, although they contain a forward slash before the character.
  • The <!DOCTYPE html> delcaration tells the browser what document type it should expect when it is opened.
  • The <html> element indicates that anything that is coded between its opening and closing tags are HTML code.
  • The lang attribute tells the browser the language for the webpage, and = "en" defines the language as English. This is what is looks like within the code: <html lang="en">
  • The <section> elements are semantic elements that define a section in a document that contains thematic content.
  • The id attribute identifies elements and selects them.

Head

  • The <head> element contains the metadata for our web page application.

    • Here are some examples of metadata tags: Title, Style, Base, Link, Meta, and Script.

  • The <title> element defines the title for a webpage. It is either shown in the top of the browser or on the tab of that page.
  • The <style> element defines the CSS styles associated with a website.
  • The <base> element defines the base URL for a webpage.
  • The <link> element connects an external resource to the HTML document.
  • The <meta> element defines metadata such as the character set, description, keywords, author, and viewport.
  • The <script> element embeds client-side scripts in the HTML document.

Body

  • The <body> element represents the visible content shown to the user.
  • Child elements are elements that are nested inside other elements.
  • Parent elements are elements that holds all the children. Parent elements can have multiple children elements. The <body> element is an example of a semantic element.
  • The <br> element creates a line of empty space, or a line break, between two blocks of content.

Semantic markup

  • Semantic HTML elements make HTML easier to read by other developers and the browser. It also provides accessibility to a variety of users, including those with screen readers.
    It is best to use semantic elements whenever possible.
  • The <div> tag creates a division in the page.
  • The <span> tag creates an inline division.
  • Strong - Text inside these <strong>strong opening and closing tags</strong> will appear in bold.
  • Emphasis - Text inside these <em>emphasis opening and closing tags</em> will appear in italic.

Structural markup

  • Headings - There are six different levels of headings. <h1> is used for main headings and is the largest. While the heading number increases by 1, the size of the heading decreases,
    until you reach <h6>, the smallest heading.

    • <h1>h1</h1> will return h1 as the header. h1 should only appear once on a page.

    • <h6>h6</h6> will return h6 as the header

  • Paragraphs - <p>A paragraph is one or more sentences combined into a larger unit. They use these opening and closing tags.</p>
  • Bold - <b>Text inside these bold opening and closing tags will appear bold.</b>
  • Italic - <i>Text inside these italic opening and closing tags will appear italic.</i>
  • Superscript - October 20<sup>th</sup> will return October 20th
  • Subscript - H<sub>2</sub>O will return H2O

Lists

  • Ordered lists
    1. Ordered lists start with the following opening and closing tags for the <ol> element: <ol></ol>
    2. In this list, use the <li></li> opening and closing tags to add items to the list
    3. Items in this list will be ordered by numbers. There should be 3 items on this list, with this being item number 3.

  • Unordered lists
    • Unordered lists start with the following opening and closing tags for the <ul> element: <ul></ul>
    • In this list, use the <li></li> opening and closing tags to add items to the list
    • Items in this list will be unordered and will contain a circle bullet point by default.

Image

<img> elements contain information about images that are displayed on the webpage. This element can contain various attributes.

  • The 'src' attribute defines the location of the image.
  • The 'alt' attribute contains a text string that describes the appearance and functionality of the image. This is important for screen readers and other assistive technologies. It can also improve search engine rankings for the website.
  • <img src="file" alt="description">

Anchor

<a> element creates links to the same webpage or other webpages.

  • href attributes point to the URL for the link.
  • <a href="URL">

Link to CSS

The HTML <link> element creates the connection to the external file.

  • <link rel="stylesheet" href="./assets/style.css">
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource. The href attribute is using a relative path. This means the location is derived from the current location (in this case, index.html). To locate the style sheet from the current location, we must look in the assets folder, hence assets/style.css.

Footer

<footer> element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.


CSS

Basics

  • CSS Rule:

    • Div {color: blue; text-align: justify}

    • Div is a selector
    • Selectors define the element or attributes to which the rules, or declarations, will apply
    • Everything in the curly brackets is the declaration bracket
    • color: blue; AND text-align; justify are both declarations
    • Declarations contain two important components: the CSS property we want to apply and the value of the property
    • color: AND text-align: are both properties
    • blue; AND justify; are both values

  • There are three ways to style a webpage using CSS:

    • inline CSS:

      • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only. They're useful for quick and permanent changes but are less flexible than external and internal style sheets, because each inline style must be edited individually whenever you make a design change.


      <h1 style="color: red;">Prework Study Guide</h1>


    • internal CSS style sheet:

      • Internal style sheets contain CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code


      <head>
      <style>
      h1 {
      color: red;
      }
      </style>
      </head>


    • external CSS style sheet

      • This is the most common way to style a webpage. This allows developers to keep all their CSS rules in a separate file. When using an external style sheet, we link the file using a <link> element placed in the HTML's <head>, as shown in the following code:


      <head>
      <link rel="stylesheet" href="./assets/style.css" />
      </head>


  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The CSS box model refers to how every HTML element is essentially a box that consists of margins outside the element, and the element's borders, padding, and content.
  • text-align;, a positioning property, is one way to center a title or heading on a webpage.
  • The <img> selector assigns propery values to any image element in the markup
  • The display property is assigned with the block value, which assigns each image to start on a new line

Class attributes

  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector. Then the class attribute can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule.


    • .card {
    width: 80%;
    margin: 40px auto;
    border: 5px solid gray;
    box-shadow: 5px 10px #888888;
    }


  • The period preceding card designates this as a class selector. Any HTML element assigned to the class, card, will gain these property assignments from the CSS
  • The first declaration addresses the width of the element at 80% of the available width
  • The margin property assigns 40px to the top and bottom, and the auto value to the left and right
  • The border property assigns a border with a width of 5px that is solid and gray
  • The box-shadow CSS property adds shadow effects around an element's frame. You can set multiple effects separated by commas. A box shadow is described by X and Y offsets relative to the element, blur and spread radius, and color. In our example, the X offset is 5px and defines the width or extent of the shadow on the X axis (or to the right of the box); the Y offset is 10px and defines the extent on the Y axis (or just below the box). We haven't defined the blur or spread radius here, but you'll learn more about this in time. We've used an HTML hexadecimal code to define the color

Git

Basics

  • Git is a tool that allows developers and others to use version control
  • GitHub is one of many web interfaces for using Git
  • A repository (repo) is a folder in which all files and their version histories are stored
  • A branch is a version of the repo that allows work without affecting other branches. Repos may have many branches for different possible changes being tested or considered, along with a default branch that serves as the sourch of truth
  • A fork is a new repository that inherits from a parent "upstream" repo. Is is used to suggest changes to an "upstream" public repo by someone who doesn't have access to edit in the repo's home org
  • Markdown (.md) is a way to write content that converts plain text to formatted text
  • Commit changes is a saved record of a change made to a file within the repo
  • A pull request (PR) is a request for changes made to a branch to be pulled into another branch. Allows multiple users to see, discuss and review work being suggested
  • After a pull request is approved, the commit will be merged from one branch to another
  • Issues allow users to report issues or bugs and track progress of assigning the fix for the issues

Git commands

  • git status checks what branch we are currently on
  • git checkout -b branch-name creates a new branch and switches to it
  • git add -A adds changes to the current working branch
  • git commit -m "enter message" creates a commit that captures a snashot of the currently staged changes
  • git pull origin main makes sure the local branch is in sync with the base branch in GitHub
  • git push origin 'branch the code is in' pushes the changes to the remote branch

For loops

  • For Loops includes an iterator variable that usually appears in all three expressions. The iterator variable is initialized, checked against the stopping condition, and assigned a new value on each loop iteration.

    • for (let counter = 0; counter < 4; counter++) {
    console.log(counter);
    }


  • Contains 3 expressions, separated by ; inside the parentheses:
    1. an initialization starts the loop and can also be used to declare the iterator variable (let counter = 0)
    2. a stopping condition is the condition that the iterator variable is evaluated against - if the condition evaluates to true the code block will run, and if it evaluates to false the code will stop. (counter < 4)
    3. an iteration statement is used to update the iterator variable one each loop (counter++)
